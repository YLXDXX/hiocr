# HiOCR

这是一个基于 Qt6 开发的桌面光学字符识别（OCR）工具，专注于屏幕截图识别、数学公式渲染以及与外部程序的集成。该程序设计精良，支持 Wayland 环境，并提供了完善的配置管理和服务管理功能。

## 1. 项目概述

HiOCR 是一款运行在 Linux 桌面环境下的 OCR 识别工具。它通过调用 HTTP 接口与识别服务交互，能够将截图中的文字、数学公式、表格等内容转换为 Markdown 格式。

**核心特性：**
*   **多模态识别**：支持文字、公式、表格等多种类型的识别。
*   **Markdown 与 LaTeX 渲染**：内置强大的 WebEngine 渲染器，支持 Temml 数学公式渲染，支持多种数学字体切换。
*   **智能复制**：提供针对纯公式和混合内容的多种复制策略，支持自定义行间公式环境。
*   **外部程序集成**：支持配置外部脚本/程序，在复制前对文本进行预处理。
*   **服务管理**：内置本地识别服务生命周期管理（自动启动、空闲检测、自动关闭）。
*   **跨平台兼容**：优先支持 Wayland (XDG Portal)，兼容 X11，针对 KDE 桌面环境提供原生全局快捷键支持。

## 2. 功能模块详解

### 2.1 图像输入与截图

程序提供多种图像输入方式，满足不同场景需求：

*   **屏幕截图**：
    *   **XDG Portal 支持**：在现代 Linux 桌面环境（特别是 Wayland）下，优先使用 `org.freedesktop.portal.Screenshot` 接口获取屏幕快照，无需 root 权限。
    *   **区域选择器**：截图后弹出半透明选区窗口（`ScreenshotAreaSelector`），用户可框选需要识别的区域。选区逻辑经过高 DPI 优化，能正确处理不同缩放比例的屏幕。
    *   **回退机制**：若 Portal 不可用，自动回退到 Qt 原生截图方式。
*   **剪贴板粘贴**：支持 `Ctrl+V` 快捷键粘贴剪贴板中的图片。
*   **文件打开**：通过文件对话框加载本地图片文件。
*   **命令行传参**：支持命令行传入图片路径直接加载。

### 2.2 识别引擎与服务交互

*   **HTTP API 兼容**：网络模块（`NetworkManager`）设计为兼容 OpenAI Chat Completion API 格式。用户可配置服务器 URL 和请求参数。
*   **动态参数配置**：支持通过设置界面修改请求参数（如 `temperature`, `max_tokens` 等），以 JSON 格式合并到请求体中。
*   **本地服务管理（`ServiceManager`）**：
    *   **自动启动**：检测到服务未运行且配置了启动命令时，自动拉起服务。
    *   **空闲超时**：当程序无活动超过设定时间（默认 10 分钟），自动终止后台服务以节省资源。
    *   **进程组管理**：在 Linux 下使用 `setpgid` 和 `kill(-pid)` 技术，确保能彻底清理服务进程及其子进程。

### 2.3 结果渲染与显示

*   **Markdown 渲染**：使用 `QWebEngineView` 加载 HTML 模板，通过 `QWebChannel` 与 JavaScript 交互。
*   **数学公式支持**：
    *   集成 **Temml** 数学公式渲染库。
    *   **字体切换**：支持 Latin Modern、Asana Math、STIX Two、Libertinus、Noto Sans 等多种数学字体。
    *   **实时预览**：左侧显示渲染后的富文本，右侧显示 Markdown 源码，两者同步滚动与更新。

### 2.4 智能复制与导出

程序提供了非常精细化的复制功能（`MarkdownCopyBar`），这是本软件的一大亮点：

*   **内容类型自动检测**：
    *   **单公式模式**：识别结果为 `$...$` 或 `$$...$$` 包裹的纯公式。
    *   **混合内容模式**：识别结果包含文字、表格或多段公式。
*   **三种复制策略**：
    1.  **原样复制**：复制 Markdown 源码（纯公式模式下会自动剥离包裹符号）。
    2.  **行内公式**：仅对单公式有效，强制转换为 `$...$` 格式。
    3.  **行间公式**：支持将公式包裹在用户设定的环境中（如 `\begin{equation}...\end{equation}` 或 `$$...$$`）。对于混合内容，会替换源码中所有的 `$$` 为设定的环境。
*   **外部程序处理**：
    *   在复制前可自动调用外部命令。
    *   通过标准输入传递文本，从标准输出获取处理后的文本。
    *   这允许用户自定义管道（如格式化、拼写检查、翻译等）。

### 2.5 快捷键系统

支持多层级的快捷键实现，以适应不同的桌面环境：

*   **KDE 原生支持**：若检测到 KDE 环境，使用 `KGlobalAccel` 注册全局快捷键，集成度高。
*   **Wayland Portal 支持**：在非 KDE 的 Wayland 环境下，使用 `org.freedesktop.portal.GlobalShortcuts` 注册全局快捷键。
*   **本地快捷键**：作为保底方案，仅在应用获得焦点时生效。
*   **可配置项**：支持配置截图、文字识别、公式识别、表格识别及外部处理调用的快捷键。

## 3. 软件架构

项目采用分层架构，逻辑清晰，易于扩展。

*   **入口层 (`main.cpp`)**：负责单实例检测、命令行解析、信号处理（SIGINT）及消息日志过滤。
*   **控制层 (`AppController`)**：核心控制器，协调 UI、截图、识别、服务管理等模块的交互。所有业务逻辑流转在此处进行。
*   **UI 层 (`MainWindow`)**：主窗口布局，包含图片预览、Markdown 渲染器、源码编辑器、提示词栏和复制工具栏。
*   **管理器层**：
    *   `ScreenshotManager`: 处理截图逻辑。
    *   `RecognitionManager`: 管理识别请求、防抖、错误重试。
    *   `NetworkManager`: 处理 HTTP 网络通信。
    *   `SettingsManager`: 单例模式，管理配置的读取与持久化。
    *   `ServiceManager`: 管理本地 OCR 服务进程。
    *   `ShortcutHandler`: 统一管理快捷键的注册与触发。

## 4. 用户配置项

通过“设置”对话框，用户可以自定义以下内容：

*   **服务器**：识别服务的 URL 地址。
*   **提示词**：预设的文字识别、公式识别、表格识别提示词。
*   **快捷键**：自定义各项功能的快捷键组合。
*   **显示设置**：
    *   行间公式环境（`$$` 或 `equation` 等）。
    *   数学渲染字体。
*   **行为设置**：
    *   是否自动使用上次的识别类型。
    *   截图后是否自动识别。
    *   识别完成后是否自动复制。
    *   复制前是否自动调用外部程序。
*   **服务管理**：
    *   是否自动启动服务。
    *   服务启动命令。
    *   空闲超时时间。
*   **高级设置**：直接编辑 JSON 格式的请求参数。

## 5. 命令行接口

支持在终端中直接调用，方便集成到脚本中：

*   `hiocr`: 启动主程序（若已运行则激活窗口）。
*   `hiocr -i <image_path>`: 启动并加载指定图片。
*   `hiocr -i <image_path> -r "<result_text>"`: 启动并直接显示指定的识别结果（静态展示模式，不触发 OCR 服务）。
*   `hiocr --verbose`: 开启详细调试日志输出。

## 6. 单实例运行机制

程序通过 `QLocalServer` 和 `QLocalSocket` 实现单实例运行。
*   首次启动时，创建本地 Socket 服务端。
*   再次启动时，尝试连接服务端，若连接成功，则将命令行参数（JSON 格式）发送给主进程，随后新进程退出。主进程接收消息后弹出窗口并处理请求。

## 7. 技术依赖

*   **Qt 6**: Core, Widgets, Network, WebEngineWidgets, WebChannel, DBus.
*   **KF6 (可选)**: GlobalAccel (用于 KDE 全局快捷键).
*   **第三方前端库**: Marked.js (Markdown 解析), Temml (LaTeX 渲染), Highlight.js (代码高亮).

该软件通过模块化的设计，将复杂的 OCR 识别流程与桌面环境深度集成，特别是在 Wayland 和 KDE 支持上的细节处理（如 Portal 截图、全局快捷键），使其成为 Linux 平台上一款功能强大且用户体验优秀的 OCR 工具。
