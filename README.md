# HiOCR

一款 Linux 桌面上的智能截图识别工具，能把屏幕上的**文字、数学公式、表格**一键转成可编辑的文本。

![Screenshot](Screenshot.png)

## ✨ 它能做什么

- **截图识别**：框选屏幕任意区域，自动识别其中的内容
- **三种模式**：文字识别、公式识别、表格识别，各有专属提示词
- **实时显示**：支持流式输出，识别结果边出边看
- **高质量公式渲染**：基于 Temml 引擎，支持多种数学字体，公式显示效果接近 LaTeX 排版
- **智能复制**：自动判断内容类型，一键复制为行内公式 `$...$` 或行间公式 `$$...$$`
- **外部脚本处理**：可配置脚本对复制内容做二次加工（去换行、格式化等）
- **多服务管理**：同时配置本地模型和远程 API，一键切换，自动故障转移
- **静默模式**：窗口不弹出，悬浮小球通知识别状态，右键查看结果
- **历史记录**：自动保存识别历史，随时回溯查看

## 🚀 快速开始

### 安装依赖

- CMake ≥ 3.20
- C++17 编译器（GCC 或 Clang）
- Qt6 组件：Core、Gui、Widgets、Network、WebEngineWidgets、WebChannel、DBus、Sql
- 可选：KF6GlobalAccel（KDE 桌面下更好的全局快捷键支持）、KF6WindowSystem （KDE 桌面下更好的窗口设置）

### 编译安装

```bash
git clone https://gitee.com/ylxdxx/hiocr
cd hiocr
mkdir build && cd build
cmake ..
make
sudo make install
```

### 首次使用

1. 启动后在设置中填写你的识别服务地址（兼容 OpenAI API 格式的服务均可）
2. 按下截图快捷键（默认 `Ctrl+Alt+M` 为文字识别截图）
3. 框选屏幕区域，等待识别结果
4. 点击底部的「内容」「行内」「行间」按钮复制你需要的内容

## 📖 功能详解

### 截图与输入

| 方式 | 说明 |
|------|------|
| 快捷键截图 | 按下快捷键后框选屏幕区域 |
| 粘贴图片 | 在图片区域按 `Ctrl+V` 粘贴剪贴板中的图片 |
| 打开文件 | 菜单栏 → 文件 → 打开图片 |
| 命令行传入 | `hiocr -i /path/to/image.png` |

截图支持 Wayland（通过 XDG Portal）和 X11，区域选择器自动适配高 DPI 屏幕。

### 识别服务

HiOCR 本身不包含识别模型，它是一个**客户端**，需要连接兼容 OpenAI Vision API 格式的服务。

**你可以使用的服务类型：**

- 🌐 **远程 API**：如通义千问、DeepSeek、Gemini 等，填入地址和 API Key 即可
- 🖥️ **本地模型**：如 llama-server、Ollama、vLLM 等，HiOCR 可以帮你自动启动

**服务管理功能：**

- 添加多个服务，在工具栏的下拉框中一键切换
- 每个服务可以单独配置地址、API Key、模型名称和提示词
- 支持两种运行模式：「仅保留一个」切换时自动停旧启新，节省内存；「并行运行」零延迟切换
- 设置「默认本地服务」后，远程连接失败时可以自动切换并启动本地服务

### 复制选项

识别结果下方有三个复制按钮：

- **内容**：复制原始识别文本（纯公式会自动去掉 `$` 包裹）
- **行内**：复制为行内公式格式 `$...$`
- **行间**：复制为行间公式格式，可在设置中选择 `$$...$$`、`\begin{equation}` 或 `\begin{align}` 环境

如果识别结果是混合内容（文字+公式），「行间」按钮会把其中的 `$$...$$` 部分统一转换为你选择的环境格式。

### 外部脚本处理

如果你需要对复制内容做进一步处理（比如去换行、转换格式），可以为不同类型的内容分别配置处理脚本：

- **文字处理**：处理普通文本识别结果
- **公式处理**：处理公式识别结果
- **表格处理**：处理表格识别结果
- **纯公式处理**：优先级最高，当结果仅包含单个公式时使用

> 在调用外部脚本时，会给脚本传一个参数 `--mode`，将当前识别服务的名称传入，可根据名称针对特定模型结果进行特殊处理

脚本通过标准输入接收文本，处理后的标准输出会写入剪贴板。例如：

```python
#!/usr/bin/env python3
import sys
text = sys.stdin.read()
print(text.replace('\n', ' '))  # 去掉换行
```

在设置中填入 `python3 /path/to/script.py`，然后在工具栏的「脚本」区域勾选对应类型即可。

每种脚本还可以绑定快捷键，方便手动触发。

### 静默模式与悬浮球

开启静默模式后，截图识别时主窗口不会弹出，而是通过通知方式告知状态：

- **系统通知**：使用系统托盘通知
- **悬浮球**：屏幕上显示一个小球，不同状态有不同颜色和动画
  - 🔵 蓝色旋转 = 正在识别
  - 🟢 绿色 ✓ = 识别完成
  - 🔴 红色 ✗ = 识别失败
  - **左键点击**：触发截图
  - **右键点击**：打开主窗口查看结果
  - **右键拖动**：移动悬浮球位置

悬浮球的大小、自动隐藏时间、是否始终显示都可以在设置中调整。

### 历史记录

识别结果会自动保存到本地数据库，包括截图和识别文本。你可以：

- 按 `Ctrl+H` 或菜单 → 文件 → 查看历史记录
- 浏览历史条目，双击加载到编辑器
- 删除单条记录或清空全部历史
- 在设置中调整保存条数上限（默认 100 条）

### 公式渲染与字体

HiOCR 使用 Temml 引擎渲染数学公式，支持以下字体：

| 字体 | 风格 |
|------|------|
| Latin Modern | 经典 LaTeX 风格（默认） |
| Asana Math | 圆润现代 |
| STIX Two | 学术出版常用 |
| Libertinus | Linux Libertine 配套 |
| Noto Sans | Google 无衬线体 |
| Local | 使用系统默认字体 |

渲染区和源码编辑区的字体大小都可以在设置中单独调整。

### 图片查看器

左侧的图片查看器支持四种显示模式：

- **整页**：适应窗口大小
- **同宽**：宽度撑满
- **同高**：高度撑满
- **原大**：1:1 像素显示

按住 `Ctrl` 滚轮可以自由缩放。右键菜单可以复制当前图片。

### 快捷键

所有快捷键都可以在设置中自定义，包括：

| 功能 | 默认快捷键 |
|------|-----------|
| 文字识别截图 | `Ctrl+Alt+M` |
| 强制中断识别 | `Ctrl+.` |
| 截图 / 公式识别 / 表格识别 | 可自定义 |
| 文字/公式/表格/纯公式脚本 | 可自定义 |

在 KDE 桌面下会使用 KGlobalAccel 原生注册，Wayland 下使用 XDG GlobalShortcuts Portal，X11 下使用 Qt 本地快捷键。

### 命令行参数

```bash
hiocr                          # 启动程序（单实例，已有实例时激活窗口）
hiocr -i image.png             # 启动并加载图片
hiocr -i image.png -r "结果"   # 加载图片并直接显示结果（跳过识别）
hiocr --verbose                # 输出详细调试日志
```

## ⚙️ 设置一览

设置文件位于 `~/.config/hiocr/hiocr.conf`，所有选项都可以在图形界面中修改。

| 分类 | 设置项 |
|------|--------|
| 识别服务 | 服务列表、切换策略、空闲超时、默认本地服务 |
| 全局默认 | 服务器地址、API Key、模型名称、提示词 |
| 快捷键 | 截图、文字/公式/表格识别、中断、各类脚本触发 |
| 显示 | 行间公式环境、数学字体、渲染/编辑字体大小 |
| 行为 | 自动识别、自动复制、自动脚本处理、自动启动服务 |
| 静默模式 | 开关、通知方式 |
| 悬浮球 | 大小、位置、自动隐藏时间、始终显示 |
| 历史记录 | 开关、条数上限 |
| 脚本处理 | 文字/公式/表格/纯公式 命令和快捷键 |
| 高级 | 请求超时、自定义请求参数（JSON） |

## 📋 常见使用场景

### 场景一：使用远程 API

1. 打开设置 → 识别服务管理 → 点击「添加」
2. 填写服务名称、API 地址（如 `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions`）
3. 填入 API Key 和模型名称（如 `qwen-vl-plus`）
4. 保存后，在工具栏选择该服务即可使用

### 场景二：本地模型自动启动

1. 添加一个本地服务，填写启动命令（如 `ollama serve` 或 `llama-server -m model.gguf --port 8080`）
2. 点击「设为默认启动」
3. 勾选设置 → 行为 →「远程连接失败自动启动本地服务」
4. 当远程连接不上时，程序会自动启动本地服务并重试

### 场景三：截图后自动处理复制

1. 编写处理脚本
2. 在设置 → 复制前外部程序处理中填入命令
3. 勾选工具栏「脚本」→ 对应类型
4. 识别完成后复制，内容会自动经过脚本处理

### 场景四：静默使用不扰人

1. 勾选设置 → 行为 →「启用静默模式」
2. 通知方式选「悬浮球」
3. 截图后窗口不弹出，看悬浮球颜色判断状态
4. 右键悬浮球查看结果，左键再次截图
